Web Forms

Web forms are interactive elements that allow users to input data (text, choices, selections) and submit it for processing.

Form Controls and Widgets

Controls are objects that allow user interaction:

  • Input boxes – for text strings and numeric values
  • Radio buttons – select one value from a small predefined set
  • Selection lists – select from a more extensive list
  • Check boxes – select values limited to two possibilities (yes/no)
  • Text area boxes – for multi-line text content

Widgets (HTML5 additions):

  • Spin boxes – integer values within a range
  • Slider controls – numeric values within a range
  • Calendar controls – date and time selection
  • Color pickers – choose color values

The <form> Tag

All form elements are placed inside the <form></form> tag. Key attributes:

AttributePurpose
actionURL of the server-side script that processes the form data. Defaults to current page if omitted.
methodHow data is sent: get or post. Defaults to GET.
targetWhere to display the result: _self (default), _blank, _parent, _top
nameName of the form
autocompleteEnables/disables browser autocomplete (on / off)
novalidateDisables HTML5 built-in validation

GET vs. POST Method

MethodHow It WorksWhen to Use
GET Appends form data to the URL after a ? character in a single step. Small forms with short fields; easier for beginners.
POST Sends data in a separate transmission after contacting the server. Forms with many/long fields; sensitive data (passwords, credit cards); more secure. 
<form action="/action_page.php" method="get">...</form>
<form action="/action_page.php" method="post" target="_blank">...</form>
Note: Each input field must have a name attribute or its data will not be submitted.

The <input> Element

The most important form element. Its appearance depends on the type attribute.

Common Input Types

TypeDescription
type="text"One-line text input (default width: 20 characters)
type="password"Password field — characters are masked
type="radio"Radio button — user selects only one from a group
type="checkbox"Checkbox — user selects zero or more options
type="submit"Submits the form to the action handler
type="reset"Resets all form values to their defaults
type="button"A generic clickable button

HTML5 New Input Types

These behave as type="text" in older browsers:

  • color – color picker
  • date, datetime-local, month, time, week – date/time selectors
  • email, url, tel – text with built-in format validation
  • number, range – numeric inputs
  • search – search field

Input Type Examples

<!-- Text -->
<form>
  <label for="fname">First name:</label>
  <input type="text" id="fname" name="fname">
  <label for="lname">Last name:</label>
  <input type="text" id="lname" name="lname">
</form>

<!-- Password -->
<input type="password" id="pwd" name="pwd">

<!-- Radio Buttons -->
<input type="radio" id="male" name="gender" value="male" checked>
<label for="male">Male</label>
<input type="radio" id="female" name="gender" value="female">
<label for="female">Female</label>

<!-- Checkboxes -->
<input type="checkbox" id="vehicle1" name="vehicle1" value="Bike">
<label for="vehicle1">I have a bike</label>
<input type="checkbox" id="vehicle2" name="vehicle2" value="Car">
<label for="vehicle2">I have a car</label>

<!-- Submit and Reset -->
<input type="submit" value="Submit">
<input type="reset" value="Reset">

<!-- Button -->
<input type="button" onclick="alert('Hello World!')" value="Click Me!">

<!-- Color Picker (HTML5) -->
<input type="color" id="favcolor" name="favcolor">

The <label> Element

  • Defines a label for a form element.
  • Useful for screen readers — reads the label aloud when the input is focused.
  • Clicking the label text toggles associated radio buttons or checkboxes.
  • The for attribute must match the id of the input to bind them.
<label for="fname">First name:</label>
<input type="text" id="fname" name="fname">

Other Form Elements

<select> — Drop-down List

<select name="movies" id="movies">
  <option value="Ten">Ten Things I Hate About You</option>
  <option value="Fifty" selected>Fifty First Dates</option>
  <option value="Dreams">What Dreams May Come</option>
</select>

<!-- Show 3 visible options -->
<select name="cars" size="3">...</select>

<!-- Allow multiple selections -->
<select id="cars" name="cars" size="4" multiple>...</select>

<textarea> — Multi-line Text Input

<textarea id="message" name="message" rows="10" cols="30">
The cat was playing in the garden.
</textarea>

<!-- Or size with CSS -->
<textarea name="message" style="width:200px; height:600px"></textarea>
  • rows – visible number of lines
  • cols – visible character width

<fieldset> and <legend>

Group related form fields with a visible border and caption.

<form action="/action_page.php">
  <fieldset>
    <legend>Personal Information:</legend>
    <label for="fname">First name:</label>
    <input type="text" id="fname" name="fname" value="Nicole"><br>
    <label for="lname">Last name:</label>
    <input type="text" id="lname" name="lname" value="Pascual"><br><br>
    <input type="submit" value="Submit">
  </fieldset>
</form>

Creating Effective Forms

  • Browser Constraints: Adjust text-input sizes to hint at expected input (e.g., 5-char zip codes). Use checkboxes, radio buttons, and selection lists to narrow user choices.
  • Handling Limited Displays: Structure forms into 2–3 logical sections. Avoid wide input elements that require horizontal scrolling.
  • User-Interface Considerations: Group input elements logically. Use consistent, well-written labels and prompts.
  • Creating Forms That Flow: Lead the user naturally through data entry in a logical order (name before address, etc.). Test on multiple browsers with real users.

HTML Multimedia

Multimedia on the web includes sound, music, videos, movies, and animations. Files are identified by their extension (e.g., .mp3, .mp4, .wav, .ogg).

The <audio> Element

HTML5 standard way to embed audio without a plug-in.

<audio controls>
  <source src="RememberMe.mp3" type="audio/mpeg">
  Your browser does not support the audio element.
</audio>
  • controls – displays play, pause, and volume controls
  • The <source> element lets you list multiple formats; the browser uses the first it recognizes
  • Text inside <audio></audio> is shown only in browsers that don't support the element

The <video> Element

<video width="320" height="240" controls>
  <source src="Feast.mp4" type="video/mp4">
  Your browser does not support the video tag.
</video>

<!-- Autoplay (does not work on mobile) -->
<video width="320" height="240" autoplay>
  <source src="movie.mp4" type="video/mp4">
  Your browser does not support the video tag.
</video>
Tip: Always include width and height to prevent page flickering while the video loads.

HTML Helpers (Plug-ins)

Helper applications extend browser functionality. They can be added via <object> or <embed> tags.

The <object> Element

Supported by all browsers. Embeds plug-ins, PDFs, Flash, other HTML, or images.

<!-- Embed a Flash/SWF file -->
<object width="400" height="200" data="banners.swf"></object>

<!-- Embed another HTML page -->
<object width="100%" height="500px" data="images1.html"></object>

<!-- Embed an image -->
<object data="panda.png"></object>

The <embed> Element

Supported by all major browsers (officially part of HTML5). Has no closing tag and cannot contain alternative text.

<embed width="400" height="50" src="banners.swf">
<embed width="100%" height="500px" src="images1.html">
<embed src="panda.png">

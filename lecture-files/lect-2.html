Before You Start Creating Websites

Ask yourself these questions:

  • What are the tools the professionals use?
  • What will my website look like?
  • What are the important files that I should have?

What Professionals Use

  • Computer
  • Text Editor – to write the code (e.g., Notepad++)
  • Web Browser – to test the code (Chrome, Firefox, Edge, Safari, Brave)
  • Graphic Editing Software – e.g., Gimp, Figma, Photoshop
  • Version Control System – e.g., GitHub, GitLab
  • FTP Program (optional) – e.g., Cyberduck, FileZilla
  • Automation System – for repetitive tasks such as running tests
  • Libraries & Frameworks

What Will My Website Look Like? – Web Design Principles

  • Clearly define your site's purpose.
  • Consider your audience.
  • Decide what your site needs to be a success.
  • Study sites of similar purpose.
  • Storyboard your site before you construct it.

What is HTML?

HTML (Hypertext Markup Language) is not a programming language. It is a markup language that tells web browsers how to structure the web pages you visit.

What is HTML5?

  • The newest version of HTML.
  • Incorporates all features from earlier versions, including XHTML.
  • Adds a diverse set of new tools for the web developer.

Goals of HTML5

  • Support all existing web pages — no need to revise older websites.
  • Reduce the need for external plugins and scripts.
  • Improve the semantic definition of page elements.
  • Make web content rendering universal and device-independent.
  • Handle web document errors in a better, more consistent fashion.

First Look at HTML5 Structure

DOCTYPE Declaration

<!DOCTYPE html>
Note: The DOCTYPE tells the browser which type and version of document to expect. This simplified declaration will be used for all future versions of HTML.

The <html> Element

<html lang="en">
Note: The lang attribute declares the page language. Always specify it — it assists search engines and screen readers. E.g., Spanish = "es", French = "fr", German = "de".

The <head> Section

<head>
  <meta charset="utf-8">
  <title>My First HTML5 Page</title>
  <link rel="stylesheet" href="style.css">
</head>

Basic HTML5 Web Page

<!DOCTYPE html>
<html lang="en">
  <head>
    <meta charset="utf-8">
    <title>My First HTML5 Page</title>
    <link rel="stylesheet" href="style.css">
  </head>
  <body>
    <p>HTML5 is fun!</p>
  </body>
</html>

HTML Page Structure Explained

<meta charset="utf-8">Specifies character set (supports most human languages)
<title></title>Sets the browser tab title and bookmark label

Four Key Concepts

1. Elements

All HTML pages are made up of elements. An element acts as a container for a portion of a page.

<h1>Text Here</h1>

2. Tags

Tags are HTML codes enclosed in angle brackets used to format text. An element is made up of an opening tag and a closing tag.

<p>  <!-- opening tag -->
</p> <!-- closing tag -->

Empty Elements – HTML elements with no content and no closing tag (e.g., <br> for a line break).

3. Attributes and Values

Attributes define the characteristics of an element and are placed inside the opening tag. Values describe the attribute.

<p align="center">This is a paragraph</p>
Tip: Always enclose values in quotation marks.

4. Nesting

Nesting means placing elements inside other elements. Elements must never overlap.

<!-- Incorrect -->
<p align="center"><h1>Incorrect</p></h1>

<!-- Correct -->
<p align="center"><h1>Correct</h1></p>

Other Important Rules

HTML Is Not Case Sensitive

<P> and <p> mean the same thing. However, W3C recommends lowercase, and XHTML demands it.

Comments

<!-- Add your comment here -->

Comments are not displayed in the browser (non-printing lines).

HTML5 Semantic Elements

Semantic elements clearly describe their meaning to both the browser and the developer.

  • Non-semantic: <div>, <span> – say nothing about content
  • Semantic: <form>, <table>, <article> – clearly define content

Indentation and Spacing

  • Use &nbsp; (non-breaking space) to add spaces/indents.
  • Use <blockquote> to indent a large block from both left and right margins.
  • Use <pre></pre> to retain text formatting exactly as entered (displays in monospaced font).

HTML Lists

Lists display specific information quickly and clearly. HTML provides three main types of lists:

  • Ordered list (<ol>) – numbered or sequential order
  • Unordered list (<ul>) – bullet points
  • Definition list (<dl>) – terms and their descriptions

Ordered List

Used when order matters (steps, instructions, procedures).

<ol>
  <li>Notebook</li>
  <li>Ballpen</li>
  <li>Yellow Paper</li>
</ol>

list-style-type values for ordered lists:

ValueDescription
decimalNumbers (default)
upper-alphaUppercase letters (A, B, C…)
lower-alphaLowercase letters (a, b, c…)
upper-romanUppercase Roman numerals (I, II, III…)
lower-romanLowercase Roman numerals (i, ii, iii…)

start and value attributes:

<!-- Start numbering from letter E (5th letter) -->
<ol type="A" start="5">
  <li>Item</li>
</ol>

<!-- Using start and value together -->
<ol class="loweralpha-list" start="2">
  <li>Notebook</li>
  <li value="5">Ballpen</li>
  <li>Yellow Paper</li>
</ol>

Unordered List

<ul>
  <li>Notebook</li>
  <li>Ballpen</li>
  <li>Yellow Paper</li>
</ul>

list-style-type values for unordered lists:

ValueDescription
discFilled circle bullet (default)
circleHollow circle
squareFilled square
noneNo marker

Description List

Uses <dl> (list), <dt> (term), and <dd> (description).

<dl>
  <dt>Notebook</dt>
  <dd>- a small book with blank or ruled pages for writing notes in.</dd>
  <dt>Ballpen</dt>
  <dd>- a pen that dispenses ink over a metal ball at its point</dd>
</dl>

Nested List

<ul>
  <li>Notebook</li>
  <li>Pen
    <ul>
      <li>Sign pen</li>
      <li>Ball point pen</li>
    </ul>
  </li>
  <li>Yellow Pad</li>
</ul>

Links / Hyperlinks

Links allow users to jump from one page to another. They can appear as highlighted/underlined text or as a graphic image.

The <a> Tag

<a> stands for anchor. The href attribute (Hypertext Reference) specifies the URL or file the link points to.

<a href="home.html">Back to HOME</a>

Absolute vs. Relative Links

TypeWhere It Goes
Absolute URLA resource on a different web server (full URL required)
Relative URLA resource on the same web server, using a path from the current page
Jump within pageA resource within the current page, using an element's id
<!-- Absolute link -->
<a href="http://google.com">Google</a>

<!-- Relative link -->
<a href="template.htm">Template</a>

Relative URL Rules

  • Use / to separate directories and files.
  • Use .. to go up to the parent directory.
<a href="../index.html">Olivia's Home Page</a>
<a href="hw1/business.html">Business Page</a>

Website Structure Types (UX Design)

StructureDescription
LinearEach page links to the next in a sequence. Good for long articles.
HierarchicalHome page links to several sub-pages acting as section hubs.
MixedHierarchical structure with extra cross-links between sections and back to the home page.

HTML Link Colors (Default)

  • Unvisited: underlined and blue
  • Visited: underlined and purple
  • Active (clicked): underlined and red

CSS Link States

a:link    { color: blue; }    /* unvisited */
a:visited { color: purple; }  /* visited */
a:hover   { color: orange; }  /* mouse over */
a:active  { color: red; }     /* being clicked */

The target Attribute

ValueBehavior
_blankOpens in a new window or tab
_selfOpens in the same tab (default)
_parentOpens in the parent frame
_topOpens in the full body of the window
framenameOpens in a named frame
<a href="https://www.w3schools.com/" target="_blank">Visit W3Schools!</a>

Images

Image Formats Comparison

FormatPronunciationBest Used ForKey Features
GIF"jiff" Navigation buttons, clip art, banners, solid-color images Max 256 colors; lossless compression; supports transparency, interlacing, animation
JPEG"jay-peg" Photographs and photo-realistic images Tens of thousands of colors; lossy compression; not ideal for illustrations
PNG"ping" Images needing transparency and high quality 24-bit true color; lossless; alpha channels; no animation support
WebP"web-pee" Modern web images replacing JPEG/GIF/PNG Lossy & lossless; supports transparency; ~30% better than PNG
AVIF"AY-viff" Next-gen web images >50% smaller than JPEG; HDR; transparency; animation; royalty-free

GIF Details

  • Maximum 256 colors; uses lossless compression.
  • Transparent GIF: browser ignores designated background color pixels, letting the page background show through.
  • Interlaced GIF: displays every 4th row first, giving a fuzzy preview while the rest loads.
  • Animated GIF: a single GIF89a file containing a series of images displayed in sequence.

PNG Details

  • Supports 24-bit true-color RGB, grayscale, and 8-bit palette.
  • Alpha channels (65,000+ transparency levels), gamma correction, 2D interlacing.
  • Does not support animation.

HTML Image Syntax

Images use the self-closing <img> tag with no closing tag.

<img src="url" alt="description">

The alt Attribute

Provides alternate text when the image cannot be displayed (slow connection, broken link, screen readers).

<img src="grizzly.png" alt="grizzly">
<img src="panda.png" alt="panda">

Image Size

<img src="panda.png" alt="Panda Bear" style="width:500px;height:600px;">

Images in a Subfolder

<img src="images/circle.jpg" alt="Circle" style="width:128px;height:128px;">

Images from Another Server

<img src="https://www.ue.edu.ph/manila/main/misc/ccss.png" alt="UE-CCSS">

Image as a Link

<a href="template.html">
  <img src="panda.png" alt="Lect1" style="width:42px;height:42px;border:0;">
</a>

Image Floating

<!-- Float image to the right -->
<p><img src="polar.png" alt="Ice bear" style="float:right;width:42px;height:42px;">
The image will float to the right of the text.</p>

<!-- Float image to the left -->
<p><img src="grizzly.png" alt="Grizzly" style="float:left;width:42px;height:42px;">
The image will float to the left of the text.</p>

Background Image

<body style="background-image:url('clouds.jpg')">
  <h2>Background Image</h2>
</body>

Image Maps

An image map defines clickable "hot spot" areas within a single image, each linking to different destinations.

Key Attributes

AttributePurpose
usemapAdded to <img> to associate it with a <map>
nameIdentifies the map (must match the usemap value)
shapeShape of the area: rect, circle, poly, default
coordsCoordinates defining the shape (relative to top-left corner = 0,0)
hrefURL the hot spot links to
targetWhere to open the link
titleTooltip text shown on hover
altAlternate text for accessibility

Coordinate Format by Shape

Shapecoords format
rectx1,y1,x2,y2 — top-left and bottom-right corners
circlex,y,r — center and radius
polyx1,y1,x2,y2,x3,y3,… — polygon corners

Image Map Syntax

<img src="image.jpg" usemap="#mymap" alt="Map image">

<map name="mymap">
  <area shape="rect" coords="0,0,100,100" href="page1.html" title="Area 1" alt="Area 1">
  <area shape="circle" coords="200,150,50" href="page2.html" title="Area 2" alt="Area 2">
  <area shape="default" href="default.html" alt="Default area">
</map>
